## 🎯 สรุปการเปลี่ยนแปลง (Summary)

### ✅ ฟีเจอร์ใหม่:
1. **เมนู "ประวัติคืนเงิน"** - แสดงประวัติการคืนเงินที่สำเร็จแล้วทั้งหมด
2. **หน้าแรก Dashboard** - แสดงประวัติคืนเงินล่าสุด 3 รายการ
3. **สถานะ "รอตรวจสอบ"** - หลังแนบสลิปแล้วไม่หาย รอแอดมินยืนยัน
4. **นับถอยหลังต่อไป** - แม้แนบสลิปแล้ว ยังคงนับถอยหลังจนกว่าแอดมินจะอนุมัติ

### ✅ การทำงานใหม่:
```
ผู้เล่นแนบสลิป → สถานะ "return_pending" (ไม่หาย)
→ นับถอยหลังต่อไป → แสดง "รอแอดมินตรวจสอบ"
→ แอดมินกด "ยืนยันคืนเงิน" → สถานะ "completed"
→ บันทึกในประวัติคืนเงิน → แสดงในหน้าแรก Dashboard
```

## 🔄 Flow การทำงาน (Workflow)

### ขั้นตอนที่ 1: ผู้เล่นยืมเงิน
```
1. ผู้เล่นส่งคำขอยืม → status: 'pending'
2. DEV อนุมัติ → status: 'approved', เริ่มนับถอยหลัง 24 ชม.
```

### ขั้นตอนที่ 2: ผู้เล่นแนบสลิปคืนเงิน
```
3. ผู้เล่นแนบสลิป → status: 'return_pending'
   - บันทึก: returnSlipUrl, returnSlipUploadDate
   - แสดง: "รอแอดมินตรวจสอบสลิป..."
   - นับถอยหลังต่อไป (ยังไม่หยุด)
   - ปุ่ม "แนบสลิป" → เปลี่ยนเป็น "สลิปถูกส่งแล้ว"
```

### ขั้นตอนที่ 3: แอดมินยืนยัน
```
4. DEV กด "ยืนยันคืนเงิน" → status: 'completed'
   - คำนวณ: now <= dueDate ? returnedOnTime = true : false
   - ถ้า returnedOnTime = true → user.points += 1
   - บันทึก: completedDate, returnedOnTime, approvedBy
   - ส่ง socket event: 'return-approved'
```

### ขั้นตอนที่ 4: แสดงในประวัติ
```
5. ระบบบันทึกในประวัติคืนเงิน
   - แสดงในหน้าแรก Dashboard (3 รายการล่าสุด)
   - แสดงในเมนู "ประวัติคืนเงิน" (ทั้งหมด)
   - Auto-Refresh ทุก 5 วินาที
```

## 🎭 สถานะต่างๆ (Status Codes)

| สถานะ | ความหมาย | สี | แสดงที่ |
|-------|----------|-----|---------|
| `pending` | รอแอดมินอนุมัติคำขอยืม | สีเหลือง | Admin: คำขอรออนุมัติ |
| `approved` | อนุมัติแล้ว (กำลังยืม) | สีฟ้า | Dashboard: หน้าแรก |
| `return_pending` | แนบสลิปแล้ว รอแอดมินยืนยัน | สีเหลือง | Dashboard: หน้าแรก + Admin: คลังสลิป |
| `completed` | คืนเงินสำเร็จ | สีเขียว | ประวัติคืนเงิน |
| `rejected` | ปฏิเสธคำขอ | สีแดง | ประวัติการยืม |
